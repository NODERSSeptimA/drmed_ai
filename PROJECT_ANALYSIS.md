# Анализ проекта MedAI

## Что это

MedAI — AI-ассистент для врачей, который через голосовой ввод заполняет медицинскую документацию. Начальный фокус — психиатрия, целевая клиника — «Династия-18». Цель — снизить бумажную нагрузку врача на 60–70%.

---

## Структура проекта

| Файл / Папка | Назначение |
|---|---|
| `web/index.html` | Дашборд врача (пациенты, метрики, AI-чат) |
| `web/medical-history.html` | Форма психиатрического осмотра (12 секций) |
| `web/styles.css` | Дизайн-система (цвета, типографика, компоненты) |
| `web/app.js` | JS — модалки, навигация по секциям |
| `templates/psychiatry-examination.json` | JSON-шаблон осмотра с AI-промптами для каждого поля |
| `mindmap-miro.txt` | Техническое описание и роадмап проекта |
| `pencil-new.pen` | Файл дизайна (макеты UI) |

---

## Текущее состояние: HTML-прототип высокого качества

### Что сделано хорошо

- Профессиональный UI с продуманной дизайн-системой (цвета, шрифты Fraunces/Inter/IBM Plex Mono, скругления)
- Полная форма психиатрического осмотра с реалистичными данными пациентки
- Модальные окна: карта пациента, создание нового пациента
- Навигация по секциям с IntersectionObserver
- Шаблон в JSON готов для интеграции с AI (промпты, follow-up вопросы)
- Грамотное планирование в mindmap (Next.js 15, PostgreSQL, Prisma, OpenAI Whisper + GPT-4o)

---

## Что НЕ реализовано

- **Backend** — нет сервера, API, базы данных (0%)
- **AI-интеграция** — кнопки голоса/файлов есть, но не работают (0%)
- **Авторизация** — нет (0%)
- **Экспорт в Word** — нет (0%)
- **Адаптивность** — нет media queries, на мобильных не работает
- **Валидация форм** — нет
- **Тесты** — нет

---

## Ключевые проблемы

1. **Разрыв между планом и реализацией** — в mindmap описан полноценный Next.js-стек, но существует только статический HTML
2. **Нет мобильной версии** — CSS без адаптивных breakpoints
3. **Нет accessibility** — отсутствуют ARIA-атрибуты, управление фокусом
4. **Данные захардкожены** — вся информация вшита прямо в HTML
